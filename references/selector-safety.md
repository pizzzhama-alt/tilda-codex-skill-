# Безопасные селекторы и scoped CSS

## Правила

- Самый безопасный scope для стандартного блока: `#recXXXXXX`, где `XXXXXX` - ID блока на странице.
- Для Zero Block добавить собственный CSS-класс элементу или группе, например `codex-price-badge`.
- Для повторяющихся элементов каталога использовать scope блока каталога, затем примерные внутренние селекторы.
- Не использовать глобальные `.t-btn`, `.t-title`, `.t-popup`, `.t-input`, если нет внешнего scope.
- Не утверждать, что примерный селектор универсален для Tilda.

## Формулировки для примерных селекторов

Использовать комментарии:

```css
/* Примерный селектор: проверьте класс карточки в конкретном блоке */
#rec123456 .t-store__card {
  ...
}
```

Или:

```js
// Примерный селектор: замените на селектор кнопки/карточки из вашего блока.
const cards = root.querySelectorAll('.t-store__card');
```

## Шаблон scoped CSS

```html
<style>
  /* Замените #rec123456 на ID вашего блока */
  #rec123456 .codex-target {
    box-sizing: border-box;
  }

  @media (max-width: 960px) {
    #rec123456 .codex-target {
      ...
    }
  }

  @media (max-width: 640px) {
    #rec123456 .codex-target {
      ...
    }
  }
</style>
```

## Zero Block

Для Zero Block предпочитать собственные классы:

1. Открыть элемент в Zero Block.
2. В настройках элемента добавить CSS class, например `codex-hero-cta`.
3. Использовать селектор через ID блока:

```css
#rec123456 .codex-hero-cta {
  ...
}
```

Если пользователь не дал ID блока, писать `#rec123456` как placeholder и явно просить заменить.
